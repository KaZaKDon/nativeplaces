# Native Places — ближайшие задачи пользовательской части

## 1. Жалобы

Подключить frontend-кнопку жалобы к backend:

```txt
POST /reports/create.php
GET  /reports/my.php
```

Минимально нужно:

- кнопка «Пожаловаться» на странице объекта;
- модальное окно с типом жалобы и текстом;
- отправка `place_id`, `report_type`, `message`;
- отображение моих жалоб в кабинете или отдельном блоке.

## 2. Отклик / сообщение владельцу

Проверить и довести сценарий:

```txt
POST /conversations/start.php
GET  /conversations/index.php
GET  /messages/index.php
POST /messages/send.php
```

Минимально нужно:

- кнопка связи с владельцем на странице объекта;
- создание или открытие диалога;
- корректное расположение сообщений владельца и пользователя;
- вывод даты, имени, текста сообщения;
- обработка пустого состояния.

## 3. Отзывы

Подключить:

```txt
GET  /reviews/index.php
POST /reviews/create.php
GET  /reviews/my.php
```

Минимально нужно:

- список отзывов на странице объекта;
- форма добавления отзыва;
- рейтинг и текст;
- состояние «ожидает модерации».

## 4. Карточка объекта

Проверить `PlacePage` и карточки списков:

- вывод всех основных полей;
- вывод контактов;
- вывод `booking_type` и `booking_url`;
- вывод динамических характеристик;
- вывод галереи;
- состояние без фото;
- состояние неопубликованного/недоступного объекта.

## 5. Избранное

Избранное уже подключено к API, но нужно проверить UX:

- состояние кнопки при открытии объекта;
- добавление;
- удаление;
- синхронизация с кабинетом;
- поведение для неавторизованного пользователя.

## 6. Оптимизация позже

Vite предупреждает о JS chunk больше 500 KB. Это не блокер.

Позже можно разделить маршруты через lazy loading:

```txt
React.lazy
Suspense
dynamic import
```
