# 18. Справочники

## Назначение

Справочники используются для хранения повторяющихся значений.

Это позволяет:

* избежать дублей
* упростить фильтрацию
* строить формы автоматически
* добавлять новые значения без изменения кода

---

## reference_groups

Группы справочников.

Примеры:

```txt
property_types
wall_materials
heating_types

fish_species

hunt_species

service_features
```

---

# 19. Динамические поля

## attribute_definitions

Описание полей для категорий.

Позволяет администратору создавать новые поля без изменения кода.

Пример:

```txt
Название: Наличие лодок
Ключ: boats
Тип: boolean
Категория: Рыбалка
```

---

Типы полей:

```txt
text
textarea
number
boolean
select
multiselect
date
```

---

## place_attributes

Хранит значения полей объекта.

Пример:

```txt
Объект: Пруд "Донской"

fish_species:
Щука
Сазан

service_features:
Прокат лодок
Мангал
```

# 20. Принцип хранения данных

Если значение уникально:

```txt
Площадь
Цена
Описание
Количество комнат
```

используется:

```txt
place_attributes
```

---

Если значение повторяется у многих объектов:

```txt
Тип недвижимости
Материал стен
Виды рыб
Виды охоты
Услуги
```

используется:

```txt
reference_groups
reference_values
```

---

# 21. Будущая админка

Администратор сможет:

```txt
Создавать новые справочники

Добавлять значения в справочники

Создавать новые поля

Привязывать поля к категориям

Управлять обязательностью полей
```

Без изменения:

```txt
БД
API
Frontend
```

---

# 22. Архитектурный принцип проекта

Native Places использует:

```txt
Справочники
+
Динамические поля
+
Единую таблицу places
```

Это позволяет расширять систему без создания новых таблиц под каждую категорию.

# План развития БД v2

## Связь объектов со справочниками

### Проблема v1

Сейчас таблица:

```txt
place_attributes
```

хранит значения в поле:

```txt
value
```

Для простых данных это подходит:

```txt
Площадь = 120
Комнат = 4
Цена = 5000000
```

Но плохо подходит для множественного выбора.

Примеры:

```txt
Виды рыб
Виды охоты
Услуги базы отдыха
Материалы стен
```

Объект может иметь несколько значений одновременно.

Например:

```txt
Пруд Донской

Щука
Сазан
Карась
```

---

## Решение v2

Создать отдельную таблицу:

```txt
place_reference_values
```

Структура:

```txt
id

place_id

reference_value_id

created_at
```

---

## Пример

Объект:

```txt
Пруд Донской
```

Справочник:

```txt
fish_species

1 = Щука
2 = Сазан
3 = Карась
```

Связи:

```txt
place_id = 15
reference_value_id = 1

place_id = 15
reference_value_id = 2

place_id = 15
reference_value_id = 3
```

---

## Что это даст

✓ Правильную нормализацию данных

✓ Быстрые фильтры

✓ Поиск по нескольким видам рыб

✓ Поиск по нескольким видам охоты

✓ Поиск по услугам

✓ Возможность расширения справочников через админку
