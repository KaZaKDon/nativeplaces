# Native Places — текущий статус frontend

Дата проверки: 14.06.2026  
Архив: `nativeplaces-main(2).zip`  
Backend-документ: `api(16).docx`

## Общий статус

Пользовательская часть проекта работает как React + Vite приложение и уже подключена к PHP API на `native-places.ru`.

Проверено локально:

```bash
npm ci
npm run build
npm run lint
```

Результат:

- зависимости устанавливаются без ошибок;
- production-сборка проходит;
- ESLint ошибок не показывает;
- Vite выдаёт только предупреждение о JS chunk больше 500 KB.

## Что сейчас реально подключено к backend

- авторизация;
- регистрация;
- выход;
- проверка текущего пользователя;
- профиль пользователя;
- загрузка аватара;
- карта объектов;
- страница объекта;
- избранное;
- мои объявления;
- создание объявления;
- редактирование объявления;
- динамические характеристики объявления;
- загрузка фотографий объявления;
- удаление фотографий объявления;
- выбор обложки;
- сортировка фотографий;
- маршруты пользователя;
- публичные маршруты по share-token.

## Последний завершённый блок

Последний рабочий блок — форма объявления и медиа:

- получение объявления для редактирования через `/my-places/show.php`;
- сохранение основных полей через `/my-places/update.php`;
- сохранение характеристик через `/place-attributes/save.php`;
- получение определений характеристик через `/place-attributes/definitions.php`;
- загрузка фото через `/place-images/upload.php`;
- удаление фото через `/place-images/delete.php`;
- выбор обложки через `/place-images/set-cover.php`;
- сортировка фото через `/place-images/reorder.php`.

## Что ещё не доведено в пользовательской части

- кнопка жалобы на объект на frontend ещё не подключена к `/reports/create.php`;
- создание отзывов на frontend ещё не подключено к `/reviews/create.php`;
- сообщения в кабинете требуют отдельной проверки UI и подключения к `/conversations/*` и `/messages/*`;
- карточка объекта требует отдельной проверки полноты выводимых полей;
- оптимизация размера JS chunk отложена.
