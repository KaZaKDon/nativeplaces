# Родные Места — статус проекта

## Что уже сделано

### Базовая архитектура

Проект построен на React + Vite.

Стек:

- React;
- Vite;
- React Router;
- Leaflet;
- OpenStreetMap;
- localStorage demo-flow.

Основные страницы:

- `HomePage` — главная cinematic-страница проекта.
- `CategoriesPage` — страница категорий мест.
- `MapPage` — карта с объектами, фильтрами и sidebar.
- `PlacePage` — отдельная страница объекта.
- `SubmitPage` — форма добавления и редактирования места.
- `SubmitLocationPage` — выбор точки на карте.
- `AccountPage` — личный кабинет в формате дневника исследователя.

## Карта

Реализовано:

- Leaflet + OpenStreetMap.
- Маркеры объектов.
- Цвета маркеров по категориям.
- Hover marker ↔ sidebar sync.
- Active marker system.
- Автофокус карты по фильтру, поиску и выбранному объекту.
- Sidebar справа на desktop.
- Bottom sheet на mobile.
- Фильтрация по категории через URL.
- Поиск по названию, описанию, категории, тегам.
- Debounce поиска.
- Нормализация поиска `ё/е`.
- Отображение объектов из основной базы и объектов из `localStorage`.
- Переход к локально добавленному объекту по параметрам URL.
- Кнопка `Добавить место` прямо со страницы карты.
- В sidebar отображаются фото локально добавленных объектов.

## Добавление и редактирование места

Реализовано:

- Страница `/submit`.
- Режим добавления нового объекта.
- Режим редактирования существующего объекта через `/submit?edit=id`.
- Выбор основной категории.
- Выбор типа объекта внутри категории.
- Динамические поля по категории.
- Базовые поля:
  - название;
  - краткое описание;
  - полное описание;
  - адрес или ориентир;
  - контакт;
  - фотографии.
- Выбор координат через отдельную страницу `/submit/location`.
- Сохранение координат в `localStorage`.
- Draft-state формы:
  - введённые данные не теряются при переходе на карту выбора координат;
  - после возврата на `/submit` данные восстанавливаются.
- Сохранение добавленного объекта в `localStorage`.
- Обновление существующего объекта в `localStorage`.
- Добавленные объекты появляются на карте в текущем браузере.
- Фото сохраняются локально в браузере в demo-режиме.
- Поддержка нескольких фотографий.
- Удаление выбранных фотографий при редактировании.
- Галерея объекта сохраняется в поле `gallery`.

## Личный кабинет

Реализовано:

- Страница `/account`.
- Кабинет оформлен как дневник исследователя.
- Фон рабочего стола.
- PNG-книга как визуальный shell на desktop.
- Бумажная мобильная версия кабинета.
- Внутренние вкладки без перехода на отдельные маршруты.
- Разделы:
  - `Мои места`;
  - `Избранное`;
  - `Сообщения`;
  - `Маршруты`;
  - `Архив`;
  - `Настройки`.
- Вкладка `Мои места` читает объекты из `localStorage`.
- Карточки объектов внутри кабинета:
  - фото;
  - слайдер фото;
  - категория;
  - статус;
  - описание;
  - локация;
  - кнопка `Редактировать`;
  - кнопка `На карте`;
  - кнопка `Удалить`.
- Кнопка `Редактировать` открывает `/submit?edit=id`.
- Кнопка `На карте` открывает объект на карте.
- Кнопка `Удалить` удаляет локальный объект.
- Вкладка `Избранное` показывает сохранённые объекты.
- Из избранного можно убрать объект.
- Исправлен мобильный скролл кабинета.

## Что пока работает в demo-режиме

- Новые места сохраняются только в `localStorage`.
- Редактирование работает только для локальных объектов.
- Фото сохраняются локально в браузере через base64.
- Избранное хранится в `localStorage`.
- Сообщения автору пока не отправляются.
- Разделы `Сообщения`, `Маршруты`, `Архив`, `Настройки` пока являются заготовками.
- Авторизация пока не реализована.
- Модерация пока не реализована.
- Backend пока не подключён.

---

## Что важно учитывать

- `localStorage` подходит только для demo-режима.
- Base64-фото быстро увеличивают объём данных в браузере.
- Для production нужно:
  - backend;
  - загрузка фото на сервер;
  - хранение URL фотографий;
  - авторизация;
  - база данных;
  - модерация объектов;
  - серверная обработка сообщений.
