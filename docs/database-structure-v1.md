# Native Places — Структура базы данных v1

## Общая информация

База данных:

```txt
vnuko1796_nativeplaces_dev
```

Система использует одну базу данных MySQL, внутри которой располагаются все таблицы проекта.

---

# Пользователи и доступ

## roles

Роли пользователей.

Назначение:

* пользователь (user)
* модератор (moderator)
* администратор (admin)

---

## users

Учетные записи пользователей.

Назначение:

* регистрация
* авторизация
* управление профилем
* связь с объектами пользователя

Связи:

```txt
users → places
users → payments
users → notifications
users → favorites
users → routes
```

## places

Главная таблица проекта.

Содержит все объекты системы.

Примеры:

* дом на продажу
* квартира в аренду
* база отдыха
* платный пруд
* охотхозяйство
* заповедник

Назначение:

* хранение основных данных объекта
* координаты
* публикация
* модерация
* срок размещения

Статусы:

```txt
pending
published
rejected
expired
closed
archived
```

Тип публикации:

```txt
free
paid
```

# Модерация

Логика публикации:

```txt
Пользователь
↓
Создание объекта
↓
status = pending
↓
Проверка модератором
↓
published / rejected
```

---

# Срок публикации

Бесплатное размещение:

```txt
14 дней
```

Перед окончанием:

```txt
уведомление пользователю
```

Если не продлено:

```txt
status = expired
```

Платное размещение:

```txt
оплата
↓
публикация
↓
продление через оплату
```
